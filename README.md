# KikakuHub

## サービス概要
KikakuHubは、RUNTEQ内で「イベントを開催したいがテーマが決められない」人のために、参加者のニーズ（やりたいテーマ）を集めて可視化し、開催に繋げるテーマ決め支援サービスです。  
投票・コメント・参加可能時間の集計を通じて「開催されやすいテーマ」と「開催しやすい時間帯」を把握し、最終的にRUNTEQ公式のイベント作成ページへ誘導します。

## 想定されるユーザー層
- RUNTEQ内でイベントを開いてみたいが、テーマが決められず着手できない学習者（主催者候補）
- 参加者として「こういうテーマのイベントが欲しい」を発信したい学習者
- 技術系（講習・ハンズオンなど）と交流系（交流会・非技術LTなど）の両方に参加したいが、生活スタイルにより参加可能時間が異なる学習者

## サービスコンセプト
RUNTEQにはイベント作成ページが既にある一方で、「イベントを作りたいがテーマが決められない」「参加者の需要が分からず開催に踏み切れない」といった課題が残りやすいです。  
KikakuHubはイベントそのものの作成・管理ではなく、企画（テーマ）決めと需要の可視化にフォーカスします。

- 参加者のニーズ（テーマ候補）を投稿・投票・コメントで集め、開催しやすいテーマを浮かび上がらせる
- 技術系・交流系で異なる参加可能時間をカレンダーで登録できるようにし、テーマごとに参加可能時間の集計を表示する
- 「開催されないテーマが一覧に溜まると離脱につながる」問題を避けるため、一定期間経過したテーマは一覧から退避（アーカイブ）して見通しを保つ
- 実際のイベント作成・登壇情報の詳細入力はRUNTEQ公式側に委ね、KikakuHubは「開催のきっかけ」を作る導線に徹する

差別化ポイントは、イベント管理ではなく「企画（テーマ）決め」と「開催しやすい時間帯の可視化」に絞ることで、主催者候補の心理的・実務的ハードルを下げ、開催率を上げる点です。

## 実装を予定している機能

### MVP
- 会員登録（合言葉による参加制限）
  - ニックネーム（日本語OK・大文字小文字を許容）
  - 何期生（integer）
  - メールアドレス
  - パスワード
  - 合言葉（共有キー1個。環境変数 `INVITE_KEY` で照合し、DBには保存しない）
- ログイン（Sorcery）
  - ユーザー名（ニックネーム）またはメールアドレス + パスワード
- テーマ（ニーズ）投稿
  - カテゴリ（技術 / 交流 / その他）
  - 技術系の形式（講習 / ハンズオン など）
  - 概要（本文）
  - 登壇募集の有無（テーマ/イベントごとに設定）
- テーマ一覧・詳細
  - カテゴリで絞り込み
  - 一定期間経過したテーマは通常一覧に表示しない（アーカイブへ）
- 投票（賛同）
  - テーマへの需要を数で可視化
- 参加・登壇希望（意思表示）
  - テーマ（イベント）ごとに「登壇者を募るかどうか」を設定可能
  - 登壇希望は「登壇してみたい」ボタンを押すだけ（必須入力なし。登壇詳細は公式側で扱う）
  - 参加 ↔ 登壇希望は同一レコードの `role` を切り替える（1人につき登録は1件）
- プロフィール
  - 参加可能時間の登録（カレンダー）
    - 技術系用と交流系用を分けて登録できる
    - その他カテゴリは登録不要・集計もしない
- 参加可能時間の集計表示（テーマ詳細）
  - テーマが技術なら技術用の参加可能時間を集計表示
  - テーマが交流なら交流用の参加可能時間を集計表示
  - テーマがその他なら集計表示しない
- RUNTEQ公式イベント作成ページへの導線
  - テーマ詳細から「このテーマでイベントを立てる」導線を用意し、開催に繋げる

### その後の機能
- 技術系形式の拡充（LT、輪読、レビュー会など）と形式別の表示最適化
- アーカイブの管理強化（復活、クローズ理由、運営用のモデレーション）
- テーマからRUNTEQ公式イベントへの紐付け（作成済みイベントURL登録など）
- 参加可能時間の例外登録（今週だけ参加不可、など）
- 通知（投票が一定数を超えた、登壇希望者が増えた、テーマがイベント化された等）

## 画面一覧（MVP）
- トップ：`/`
- サインアップ：`/signup/new`
- ログイン：`/session/new`
- ログアウト：`DELETE /session`
- テーマ一覧：`/themes`
- テーマ作成：`/themes/new`
- テーマ詳細：`/themes/:id`
- アーカイブ一覧：`/themes/archived`
- プロフィール：`/profile`
- 参加可能時間編集（技術/交流タブ）：`/profile/availability`
