# Issue の書き方ガイドライン

## タイトルの書き方

### 基本形式

```
[カテゴリ名] 概要説明
```

### カテゴリ名の例

| カテゴリ | 使いどころ |
|----------|------------|
| テーマ投稿 | テーマ（企画）関連の機能 |
| 参加可能時間 | 参加可能時間の登録・集計 |
| 認証 | ログイン・サインアップ・権限 |
| UI | 画面デザイン・レイアウト全般 |
| インフラ | Docker・CI・デプロイ |
| MVP | MVP段階の新機能 |
| バグ | 不具合の報告 |

### タイトルのルール

- 日本語で書く（技術用語は英語OK）
- 「何をするか」を簡潔に書く
- 「XXXを追加」「XXXを修正」「XXXを改善」のように動詞で終わる

### 良い例

- `[テーマ投稿] タイトルの文字数バリデーションを追加`
- `[参加可能時間] 集計パネルのカテゴリ切り替えを非同期化`
- `[MVP] ユーザー登録フローにニックネーム入力を追加`
- `[バグ] テーマ詳細画面で投票数が正しく表示されない`
- `[UI] パンくずリストを主要画面に導入`

### 避けたい例

- `修正` → 何を修正するのかわからない
- `テーマの件` → 具体的な内容が不明
- `Feature/65 UI improvements` → 英語混在、内容が曖昧

## 本文の構成

### 必須セクション

#### 背景 / 目的（Why）

なぜこの変更が必要なのかを説明します。

```markdown
## 背景 / 目的（Why）

- 現状の課題: テーマ一覧が増えてきたが検索手段がなく、目的のテーマを見つけにくい
- ユーザーストーリー: 参加者として、キーワードでテーマを検索したい。なぜなら一覧をスクロールして探すのが手間だから
```

#### 仕様（What）

具体的に何を実装するかを記載します。

```markdown
## 仕様（What）

- テーマ一覧画面の上部に検索フォームを追加
- タイトルと説明文を対象にキーワード検索（部分一致）
- 任意でカテゴリ（tech / community）による絞り込みも可能
- 検索結果は既存の一覧と同じカード形式で表示
```

#### 受け入れ条件（Acceptance Criteria）

完了と判断する基準をチェックリスト形式で書きます。

```markdown
## 受け入れ条件（Acceptance Criteria）

- [ ] テーマ一覧画面に検索フォームが表示される
- [ ] キーワードを入力して検索すると、該当するテーマのみ表示される
- [ ] カテゴリで絞り込みができる
- [ ] 検索結果が0件の場合、適切なメッセージが表示される
- [ ] Rubocopが通る
```

### 任意セクション

#### 実装メモ（方針）

技術的なアプローチや参考にすべき既存実装のメモです。

```markdown
## 実装メモ（方針）

- ransack は導入せず、scopeベースで実装する（MVP段階では過剰）
- `Theme.search(keyword, category:)` のようなクラスメソッドを追加
- Turbo Frame で検索結果を非同期更新する
```

#### 参考リンク

関連するissueや外部ドキュメントへのリンクです。

```markdown
## 参考リンク

- 関連issue: #42
- Rails Guides: https://guides.rubyonrails.org/...
```

## ラベルの付け方

issueには以下のラベルを組み合わせて付けます。詳細は [ブランチ命名・ラベル運用ガイド](branch-naming.md) を参照してください。

- **type ラベル**（必須: 1つ）: `type:feature` / `type:bug` / `type:refactor` / `type:docs` / `type:chore`
- **area ラベル**（任意）: `area:auth` / `area:themes` / `area:availability` / `area:ui` / `area:infra`
- **priority ラベル**（任意）: `prio:high` / `prio:mid` / `prio:low`

## チェックリスト

issue を作成する前に確認してください。

- [ ] 同じ内容のissueが既にないか検索した
- [ ] タイトルに `[カテゴリ名]` を含めた
- [ ] 背景（Why）/ 仕様（What）/ 受け入れ条件（AC）を記載した
- [ ] 適切なラベルを付けた
